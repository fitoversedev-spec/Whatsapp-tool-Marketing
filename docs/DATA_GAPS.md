| 25 | No Account-editing screen exists at all — name/city set once, never correctable. | **Resolved 2026-07-16** — `Account.name`/`city` added to the Deal Detail edit modal (`PATCH /api/deals/[id]`'s existing Account write-through block, prefixed `accountName`/`accountCity` to disambiguate from the Deal's own `title`/fields). Simplification: renaming to match another Account's name does NOT trigger a merge/duplicate-prompt — `resolveAccountId()`'s dedup only runs at create-time, not on rename. Acceptable for now; a real Account-merge feature is a bigger, separate ask. |

Also syncs `Deal.quotedValue`/`Conversation.dealValue` on the edit, but only when the edited quotation is the deal's current primary revision. |
| 27 | `DealStageHistory.note` never sent from the `/deals` closeout modal. | **Resolved 2026-07-16** — Won gets its own optional Note field (nothing else existed for it); Lost reuses the existing loss-reason Notes text as the history note too, rather than asking for the same context twice in one modal. |
